# Skill: doc

## Install

```bash
npx skills add doc
```

## Why it is useful for mo-code

`mo-code` depends on good handoff quality. The project already treats docs as operational artifacts, not decoration:

- `CHECKPOINT.md`
- `TODO.md`
- architecture and protocol docs
- implementation notes
- future onboarding docs for new agents

This skill is useful whenever the task includes documentation quality, structure, or synchronization with code.

## Use this skill when

- updating `CHECKPOINT.md`
- updating `TODO.md`
- refining architecture or protocol docs
- writing setup or onboarding material
- keeping docs aligned with implementation

## Priority

Medium-high. Recommended because this repo is explicitly multi-agent.

- `mo-code-creation`
