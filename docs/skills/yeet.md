# Skill: yeet

## Install

```bash
npx skills add yeet
```

## Why it is useful for mo-code

The product promise is "build on mobile, push to GitHub." A skill that formalizes commit, push, and PR handling is directly useful here.

This is also useful for the repo itself when agents need to publish intentional changes instead of ad hoc git usage.

## Use this skill when

- preparing a feature branch for review
- creating a draft PR
- packaging a coherent set of local changes
- making sure commit/push scope is deliberate

## Priority

Medium-high. Recommended once development starts flowing through PRs.

## Pair with

- `gh-fix-ci`
- `gh-address-comments`
